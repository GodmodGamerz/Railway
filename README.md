# Railway SSH VPS A persistent Linux VPS (Ubuntu 22.04) deployable on Railway with SSH access. ## Files - `Dockerfile` — Ubuntu 22.04 with SSH + common tools - `start.sh` — Sets password from env var, starts SSH server - `railway.toml` — Railway build/deploy config ## Deploy Steps ### 1. Push to GitHub ```bash git init git add . git commit -m "Railway SSH VPS" git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git git push -u origin main ``` ### 2. Create Railway Project 1. Go to [railway.app](https://railway.app) and log in 2. Click **New Project** → **Deploy from GitHub repo** 3. Select your repository ### 3. Set Environment Variable In Railway dashboard: - Go to your service → **Variables** - Add: `SSH_PASSWORD` = `your_secure_password_here` ### 4. Enable TCP Networking Railway needs TCP proxy for SSH: 1. Go to your service → **Settings** → **Networking** 2. Click **Add TCP Proxy** 3. Set the target port to **22** (or leave blank to auto-detect from $PORT) 4. Railway will give you a hostname + port like: `roundhouse.proxy.rlwy.net:12345` ### 5. Connect via SSH ```bash ssh vps@roundhouse.proxy.rlwy.net -p 12345 # Enter your SSH_PASSWORD when prompted ``` ## Installed Tools - git, curl, wget - vim, nano, tmux, htop - python3 + pip - nodejs + npm - sudo (passwordless for vps user) ## Notes - Data is **not persistent** across redeployments (Railway containers are ephemeral) - For persistent storage, add a Railway **Volume** in the dashboard - The `vps` user has full sudo access - Free Railway plan gives 500 hours/month
